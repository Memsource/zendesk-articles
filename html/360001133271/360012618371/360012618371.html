New Features and Improvements

Preserving Layout Changes

It has been possible for users to change the position of the different windows in the Memsource Desktop Editor for a while, but previously when the Editor was closed, the windows were restored to their default position.

With the latest release of the Desktop Editor, any changes to the layout of the windows will be saved and re-applied when the user opens the Desktop Editor again.

Support for Bi-directional Texts

Like the Web Editor, the Desktop Editor now also allows users to insert marks in target segments where the direction of the text changes from right-to-left to left-to-right and vice versa.

Included with this feature is improved support for bi-directional texts in our in-context preview; when switching between source and target languages in the in-context preview, the text direction of the preview will also be switched.

Fixed Bugs

  • When working with a Hebrew keyboard layout, several of the system keyboard shortcuts were not working properly in the previous version of the Editor. These issues have now been resolved.

  • The rendering problems of Amharic and Tigrigna texts have now been resolved.
