The YAML file filter by default imports files with the .yaml extension.

Below is a list of the file import settings that can be selected when creating a new job:

  • Use HTML subfilter
  • Convert to Memsource tags (use regexp): Matching text will be converted to Memsource tags.
  • Import specific keys only (use regexp): See our Regexp article for more details.
  • Context key *: The value that constitutes TM context (101% match) if applicable.
  • Context note *: Matching content will be displayed in the context note. For example: ../title/en.

* See Memsource Limits for YAML
