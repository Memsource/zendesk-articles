Tags represent formatting information in a segment. They are generally not used for bold, italics, underlines, superscript and subscript formatting, which are represented visually. However, for all other types of formatting, tags are used. Because tags represent formatting information in a segment, it is important to also include tags in the target, so that the formatting is preserved correctly in the exported target file.

For most file formats, you can display the content of the tags by expanding them (by pressing Ctrl+Shift+E or going to the Edit menu, clicking on Tags and Expand Tags).

Tag Types

There are two types of tags—paired and unpaired tags:

  1. Paired tags come in a pair and represent formatting that spans across a section of text. For instance, a section of text in a segment might be in a different font than the rest, or a section of text is highlighted, etc. Paired tags can be inserted via selecting the text around which the paired tags should be inserted and hitting F8.
  2. Unpaired tags can represent, for instance, the position of a footnote in a segment or can indicate a joined segment. Unpaired tags are also inserted via F8 in the Memsource Editor.

 Joined Segments tags are a special kind of unpaired tag—every time you join two segments together, the {j} tag will appear in the source and in the target, indicating where the segments were joined. It is important to keep those tags in the segment to ensure that the translated file will have the correct formatting. The {j} in the target can be placed anywhere in the translated text. The resulting text will then be formatted accordingly.

If tags are missing in the target, Memsource will do its best to auto-complete them in the background when the completed target file is being generated. But this is not always possible, so you should always add tags manually.

Quality Assurance

To check whether tags have been properly placed in all target segments, use the Quality Assurance feature in the Memsource Editor and make sure the Quality Assurance Settings in your Memsource project have been set up to check tags (Tags/Formatting and Join tags {j} must be selected).

Missing or incorrect tags can prevent Memsource from exporting a valid translated document. The error will be displayed on the project page. If this happens, open the editor, run the QA check, and fix all Tags/Formatting errors.

The Importance of Keeping Tags

It is advisable to insert tags. They should be inserted in such a way that their position and order correspond to the source as much as possible.

It is not advisable to remove tags. If tags are not inserted (or the user forgets to insert them), Memsource will insert them automatically in the background when exporting the completed target file (it will not, however, save them to the MXLIFF file). This means that Memsource will try to guess where the missing tags should be placed and will place them there automatically. The result will probably not be as good as if a human inserted the tags and so it may result in incorrect formatting in some cases. In some extreme cases, it could result in loss of some content in the translated file.

 

Note: There is a QA check that helps identify any tag-related problems quickly (e.g. missing or misplaced tags). 

Convert to Memsource Tags

In most file types, users can use the Convert to Memsource Tags option to convert certain parts of texts that should not be translated, such as [codes in square brackets], {variables}, codes containing numbers and letters (specified using regular expressions), any proper names, etc. to Memsource tags.

In this case, these will be replaced with non-paired tags, and thus, protected from translation.
Translators can use the Ctrl+Shift+E keyboard shortcut to display the content of tags in the Memsource Editor for better context.

Escaping Tags

If a number is in curly brackets, for example {1},  and needs to be kept as such and not be automatically converted into an MXLIFF tag in the Editor, it needs to be escaped: \{1\} 

Editing a Tag's Content

Sometimes, you may need to edit the content of a tag (such as the href attributes in anchor tags).

Step 1: Press CTRL+SHIFT+E or click on Edit, Tags, then Expand Tags to visualize tag content.

Step 2: In the Memsource Editor for the web, click on the tag you want to edit and start editing. In the desktop editor, click on the tag and then hit F2 to edit it.

The edited tag will be displayed in yellow color to indicate it is different from the source tag.

Shortcuts for Inserting Tags into a Target Segment 

There are two options for quickly inserting tags.

Step 1: Specify the tag number and position of the tag in the target segment using the shortcut CTRL+ALT+Tag Number (or Cmd+Option+Tag Number on a Mac). The tags can be added in any order. To insert a paired tag, you can insert them one after another, or highlight the section that should be within the paired tags and then press the keyboard shortcut together with the appropriate tag number.

Step 2: Use the shortcut CTRL+comma (for both Windows and Mac) and select tags from a list. 

Custom Tags

A user can add a custom tag or custom paired tags into the target. In the Editor's menu, select Edit, Tags, and choose Create Custom Tag or Create Custom Paired Tag and then edit the content of the tag as needed. A custom tag will be displayed as a yellow number (instead of the standard blue number).

Note: The adding of custom tags is available only for formats supporting tags in the completed file (such as HTML).
