In each project's Job section, you'll find an Edit button that allows for editing job information in single or multiple jobs at once. To edit multiple jobs at once, select all the jobs you wish to be edited and hit the Edit button.

The following information can be edited:

  • Linguist
  • Status (Please note that when set to Completed by Linguist or Delivered, any changes by Linguist users are blocked on the server.)
    • New
    • Emailed to Linguist
    • Accepted by Linguist
    • Declined by Linguist
    • Rejected by Linguist (This is only available for Ultimate and Enterprise editions. Rejected by Linguist will return the job to the previous Workflow step if, for example, a proofreader thinks the translation is not good enough and needs redoing.)
    • Completed by Linguist (Please note that changes by Linguist users are blocked on the server.)
    • Delivered (Changes made by Linguist users are blocked on the server.)
  • Due

Editing Multiple Jobs

When editing multiple jobs, you can disable certain form fields by clicking the x on the right-hand side. Doing this for any field means that the field will be hidden and will remain unchanged. For example, when you need to change the status of the jobs assigned to different Linguists, disable (or hide) the Linguist field, change the Status and Save it. The Status will then be changed for all the selected jobs while the Assigned Linguists will remain unchanged.

When multiple jobs are selected for editing, a yellow triangle with an exclamation mark will be displayed. There will also be a tooltip reminder stating that the values will be overwritten for all selected jobs.

Assigning Jobs to Linguists

To assign jobs to a Linguist, select the jobs, and click on the Edit button. Select one or more Linguists and click on Save.

Relevant Linguists

The Team, Ultimate, and Enterprise editions will help suggest relevant Linguists when available. A Linguist will pop up as relevant when some of the project and/or job attributes match the Linguist attributes. These include:

  • Source and target languages
  • Workflow step
  • Client
  • Domain
  • Subdomain
