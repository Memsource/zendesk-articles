Word and other Text documents

Memsource supports the following text document formats:

.doc .dotcm
.docx .dotm
.dot .rtf
.dotx  

The MS Word File Filter by default imports files with the following extensions:

  • .doc
  • .docx
  • .rtf
The File import options that can be selected when creating a new job are:
  • Import index entries
  • Import comments
  • Import hyperlink target (This lets you change where the link leads. For example, it would allow you to change http://google.com to http://google.de when translating to German.)
  • Minimize the number of tags (This will ignore smaller formatting differences in, for example, scanned documents.)
  • Import document properties
  • Import hidden text
  • Target font (Allows you to specify the font of the text in the Completed document.)
  • Convert to Memsource tags (Use regular expressions for converting specific text to tags— see our Regexp manual for more information.)

 

Please note that hidden content does not get imported. This means that if some content should not be imported for translation, you can mark it hidden in the MS Word file.

Should your document contain any embedded objects (for example, other files linked with your document), the following warning will be displayed: There are embedded objects within the document which have not been imported. The embedded objects can be other office documents, images or videos—these objects will not be translated. See the instructions on How to import embedded objects into Memsource.

 

PowerPoint and Slide Show Formats

For PowerPoint, Memsource supports these formats:

.ppt .potx
.pptx .pptm
.pot .potm

 

The MS PowerPoint File Filter by default imports files with the following extensions:
  • .ppt
  • .pptx

The file import options that can be selected when creating a new job are:

  • Import notes
  • Import master slides
  • Import hidden slides

 

Excel Files

The MS Excel file filter by default imports files with the following extensions:

  • .xls
  • .xlsx

 

File import options that can be selected when creating a new job:

  • Cell Flow:
    • Columns, left to right (the default option)
    • Rows, left to right (Select whether or not the content flows by rows, as it would in a Word document.)
    • Columns, right to left (This is an alternative to the first option and should be used for right-to-left languages, such as Arabic or Hebrew.)
    • Rows, right to left (This is an alternative to the second option and should be used for right-to-left languages, such as Arabic or Hebrew.)
  • Import comments
  • Import sheet names
  • Import editable texts from drawings
  • Import hidden columns, hidden rows, and hidden sheets (Please note: Hidden content does not get imported by default. If some content should not be imported for translation, mark it Hidden in the MS Excel file.)
  • Process HTML code (if your .xlsx file contains HTML tags such as <b>, <span>,  etc.):
    • Only the option Convert to Memsource tags (use regexp) is available; all other options are disabled.
  • Convert to Memsource tags (use regexp):
    • If you need to convert special text into tags use the regular expression \{[^\}]+\}
    • This can be used to convert a {variable} for example.

 

Google Docs, Sheets, and Slides

Memsource is able to read documents from Google Drive, including Google Documents. Memsource supports Docs (an equivalent to .docx), Sheets (.xlsx) and Presentations (.pptx). For more information on how to translate files stored in your Google drive, please see our Google Drive Connector page.

 

 

Htm, Html

Here are the file import settings for HTML files that can be selected when creating a new job:

  • Break tags to create a new segment
  • Convert non-HTML tags to Memsource tags
  • Import comments (This lets you import standalone comments as tags. Inline comments—comments inside a paragraph—will always be imported as tags.)
  • Preserve whitespaces
  • Exclude elements (An example of an element you might want to exclude would be a list of HTML tags separated by commas.)
  • Convert to Memsource tags (use Regexp)
  • Character entities to Memsource tags
  • Translatable meta tags (These use regular expressions to specify attributes in a tag to import their content. Please note that this is available only with attribute name="description" or name="keywords".)

 

Tag Content Visualized

The tag content of the source HTML file can be visualized via clicking Edit. These tags can be edited by pressing F2 in Memsource Editor.

 

Other File Formats

Memsource also supports .idml, .indd, .icml, .mif (version 8 and above only), .psd, and .svg files. File import options for Adobe InDesign files that can be selected when creating a new job are:

  • Extract notes
  • Simplify codes (simplifies formatting tags)
  • Extract master spreads
  • Extract locked layers
  • Extract invisible layers
  • Extract hidden conditional text
  • Keep kerning*
  • Keep tracking*
  • Replace font
  • Remove XML elements
  • Target font
    • A font name can be entered by the user. If no font name is entered, default fonts for the following languages will be used:
      • Chinese (Taiwan) - PMingLiu
      • Chinese - SimSun
      • Arabic - Arial
      • Thai - Tahoma
      • Hebrew - Arial
      • Vietnamese - Arial
      • Japanese - MS Mincho
      • Korean - Arial Unicode MS
* You can find more information about kerning and tracking on the Adobe help page.

 

Notes:

Index pages for translation are placed at the end of the imported document. In order to have a translated index page in the completed IDML file, it is necessary to open the document in Adobe InDesign and generate the index page again.

The IDML format is directly supported. INDD will be automatically converted to IDML when imported (courtesy of our partner Frontlab) and will always be exported in IDML format.

It is possible to Preview a PDF of the IDML files in Memsource Editor, but only with Frontlab Integration.

 

.mif (version 8 and above only) Adobe FrameMaker

The Adobe FrameMaker File Filter by default imports files with the .mif extension. FrameMaker files version 8 and above are supported.

File import options that can be selected when creating a new job are:

  • Extract body pages
  • Extract reference pages (pages with HTML, Headings, and MasterPageMaps page tags)
  • Extract custom reference pages
  • Extract default reference pages (pages with page tag TOC, Reference, and IX)
  • Extract master pages
  • Extract hidden pages
  • Extract hidden conditional text
  • Extract variables
  • Extract used variables
  • Extract index markers
  • Extract links
  • Extract text from XRefDef
  • Extract text from PgfNumFormat

 

For the following languages, the target font is automatically changed:

  • Chinese (Taiwan) - ZH_TW: PMingLiu
  • Chinese - ZH: SimSun
  • Japanese - JA: MS Mincho
  • Korean - KO: Arial Unicode MS
  • Thai - TH: Tahoma
