Memsource tries to determine the encoding from the file header. If this is not possible the default encoding of the format's standards is used:

  • For most cases it is UTF-8
  • iOS Strings: UTF-16,
  • Java properties: ISO-8859-1,
  • SRT: windows-1252.
  • Joomla INI: must always be in UTF-8 (It is not possible to change this with Input and/or Output Encoding settings.)

The encoding of the imported and exported files can be changed when the job is created.

Open New Job > File Import Settings >Character Set

  • Input Encoding for imported files
  • Output Encoding for exported translated file
  • Encoding of zipped file names - in case the files in your ZIP have differently encoded names
