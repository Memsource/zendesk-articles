New Features and Improvements

New Shared Project Icon

Shared Projects can now be distinguished by a reworked icon (both in the project and in the list of projects). 

 

API Call To Retrieve Default File Import Settings

The File Import Settings defined under Project Settings in Setup can be considered the default ones for any project not created via a Project Template. It is now possible to retrieve these settings using the Import Settings REST API. To find out more, please refer to our REST API Documentation.

 

Profile UUID in Systran PNMT Settings

Using the Profile field, users of Systran PNMT can now select a specialized engine to be used in Memsource.

Fixed Bugs

  • Using a Hindi spellchecker caused the Hindi font to break. This issue has now been fixed.
  • We have resolved a bug where participants in a shared scenario saw their organizations' usernames displayed for actions performed by other members of the supply chain. This has now been fixed and the appropriate user (the one who has performed the action) is now displayed. If the user belongs to a different organization, an ID is displayed instead. 
  • Jobs created by a Job Owner were incorrectly displayed in the Project Owner's profile. This bug has now been fixed.
  • Importing a ZIP file via the REST APIs will no longer result in a 500 Internal Error being returned to the requestor. 
  • When sending the login information using the Email Login button, users set as CC or BCC did not receive these notifications. This bug has now been fixed.
  • If Ignore Tag Metadata was used in combination with Penalize multiple 101% matches, an incorrect pre-translation index was given to these segments during pre-translation. These segments will no longer be pre-translated if the threshold is set to 101%. 
  • Due to a bug, it was not possible to assign TMs for language variations of the same parent language (e.g. for EN_GB and EN_US). This bug has now been fixed.
  • A font change in the Email window has been reverted back to the normal weight.
  • It was not possible to save REGEXP QA settings. This has now been fixed.
  • Automatic Email with Login Credentials is now correctly sent even for users imported from an XLSX table.
  • Automatic notifications to Linguists were sometimes sent even when the Linguist was assigned later or the status changed from Completed. This has now been resolved.
