Support tickets are categorized according to the severity with the designations P1/P2/P3/P4. These designations provide guidance and are used by the support team to process tickets according to the assigned priority.

Important

Only P1 severity requests that require an immediate response or direct help from Memsource Technical Support Agents may be processed out of turn.

Before opening a ticket, please carry out, basic First Level support, including but not limited to, basic problem identification and diagnosis.

When submitting a ticket:

  • Classify the case according to the Ticket Priority Table.

  • Do not report every issue as P1-BLOCKER.

  • Supply detailed information so the Memsource Technical Support Team can properly identify and diagnose the issue.

This speeds up the response times and facilitates the resolution of the ticket. Failure to provide detailed information or to incorrectly classify a ticket will result in support delays.

Ticket resolution time cannot be guaranteed for a number of reasons including, but not limited to:

  • Timely end-user replies.

  • Response time of a third party company.

  • Level of effort - simple tickets have a shorter implementation time than complex ones.

  • Engineering teams and resources availability.

  • Preparation time required to release a hotfix, software update, new feature, etc..

Definition

Example

P1-BLOCKER

Memsource as a whole or one of its components mentioned on the Status Page is not available and productivity has been halted. This includes situations in which custom integration with Memsource is broken and severe performance issues are experienced. The issue usually affects the majority of Memsource users and is being reported by other users.

To treat a ticket as P1-BLOCKER by Memsource Technical Support Team, the distinguishing criterion is if the problem is going to be, or already is, reported on the Status Page. Support agents provide updates on the SP every 30-60 minutes and the submitter is inform directly in the ticket upon resolution.

If the P1-BLOCKER ticket severity level is used to report a non-P1 issue, the severity of the ticket is immediately downgraded by the Memsource Support Agent.

  • Saving segments in the Memsource Editor for Web is repeatedly failing for all users and an error message keeps popping up.

  • Memsource Editor for Web is not accessible.

  • Project management features are not responding or extremely slow.

  • Memsource is down.

P2-HIGH

A ticket, other than P1, causes an inconvenience; a major feature is not working. Memsource as a whole, including all the components are available, but issues are experienced that have a direct impact on productivity.

  • The Alignment feature does not work.

  • Uploading bilingual files is failing.

  • Memsource editor for Web is not accessible via our trusted network restriction feature.

  • Widget verification emails do not work.

  • TM keeps disconnecting from TE for Moravia projects.

  • QA results in an error message.

  • WP connector doesn’t list all pages.

  • Booking -  multiple providers have access to the same job at the same time.

P3-NORMAL

A ticket, other than P1 or P2, causes an inconvenience or a moderate inconvenience; a feature or functionality does not work as expected, but a workaround is available. Memsource as a whole is operational, but the problem causes a non-negligible impact.

  • The filtering functionality is not working in Editor for Web, but works as expected in Editor for Desktop.

  • F8 shortcut inserts the first tag several times in RTL languages.

  • Complete file is not downloaded from the correct workflow in IE and Edge, however downloading works fine in Chrome/Firefox.

  • Linguist cannot connect to the TM when using the Desktop Editor.

  • Linguist cannot log in to Desktop Editor due to Editor login settings.

  • Various issues with Word Count Status bar.

  • Job notifications get mixed up in the notification email.

  • Jobs cannot be exported back to the online repository.

  • Client cannot unshare a project.

P4-LOW

A minor problem condition that does not affect Memsource operation such as general requests for advice on product usage, clarification on product documentation or release notes, requests for product enhancements, and cosmetic UI problems.

  • UI problems - overlapping tooltips.

  • Highlighting a word in the Editor for Web returns the cursor in the middle of the word.

  • How to import an XML file?

  • Allow linguist users to export workflow changes

  • Conversation bubbles in TWE do not turn blue where they should.

  • How to import a Multilingual XLS file.?

  • Wrong content imported - incorrect regexp used.

  • Incorrectly localized UI.
