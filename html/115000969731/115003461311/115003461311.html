This article is specific to the Academic Edition.

If you have any suggestions or if you need any assistance, please contact Britt Nelson.

If you've activated your Academic edition but do not know how to begin teaching with it, you're in the right place. There are three suggested ways of working with the Academic Edition that depend on how the use of translation and project management software is going to be taught. Choose the one that suits you best.

  • Translation

  • Translation and project management

  • Simulated agency

Translation

  • TMs and TBs are not shared so only the effort of the individual student creates value and is measured.

  • As many jobs as required can be added to a project. With every job performed, TMs and TBs grow to leverage earlier translation work performed by that student.

  • If there are more lecturers in your translation department and you want them to manage their own groups of students ,create a PM account for each of them.

To setup Academic Edition for translation education, follow these steps:

  1. Create a Linguist account for each student. If prepared in a spreadsheet, bulk import of users can be used.

  2. Create a project for each student. A project template may be created first and used for all students.

    The project naming convention Project [Student Name] is suggested.

  3. Create a TM and a TB for each student. If externally prepared, these can be imported to a created TM or TB.

    The naming convention TM [Student Name] and TB [Student Name] is suggested.

  4. From the project page, assign TMs and select TBs for student projects.

  5. Create jobs in the projects for each student.

  6. As students complete their assignments (jobs), change the status of the job to Completed by Linguist.

  7. Completed assignments can now be downloaded from individual student's projects.

Translation and Project Management

  • Students create and maintain their own projects, TMs, and TBs. These resources are only visible to the student who created them.

  • TMs and TBs are not shared so only the effort of the individual student creates value and is measured.

  • As many jobs as required can be added to a project. With every job performed, TMs and TBs grow to leverage earlier translation work performed by that student.

To setup Academic Edition for translation and project management education, follow these steps:

  1. Create a PM account for each student. If prepared in a spreadsheet, bulk import of users can be used.

  2. Set the following user rights for each student:

    • Create projects

    • Create project templates

    • Create TMs

    • Create TBs

    • Create users

    • Create clients, domains, subdomains

    • Create vendors

    • View data owned by the user

  3. Have students create their own projects.

    The project naming convention Project [Student Name] is suggested.

  4. Have students create a TM and a TB. If externally prepared, these can be imported to the created TM or TB.

    The naming convention TM [Student Name] and TB [Student Name] is suggested.

  5. From the project page, have students assign their TMs and select their TBs.

  6. Create jobs in the projects for each student. This can be done by yourself as the educator, or files can be sent to the students to be uploaded as jobs.

  7. As students complete their assignments (jobs), change the status of the job to Completed by Linguist.

  8. Completed assignments can now be downloaded from individual student's projects.

Simulated Agency

  • The license can be used to setup a simulated translation agency. Educators or students can create projects, TMs and TBs and assign jobs to translators, reviewers, etc.. Assigned jobs enter a specified workflow and notifications are sent to defined users.

  • It is recommended that the same text is not translated more than once in the same language combination. Translated texts are saved in the TM and no translation work will be required if replicated.

  • The administrator is able to monitor all processes going on within the license.

  • Work with any language pair that your colleagues and students speak. If there are pairs of students that speak the same foreign language, make them translator–reviewer pairs and let different pairs translate the same source text to various target languages.

  • Roles can be changed. The accounts do not have to be assigned to specific names so students may try different roles.

  • Use deadlines to inform students when assignments should be finished.

  • Lecturers from the Maastricht School of Translation and Interpreting have prepared a network of simulated agencies. Visit their website if interested.

To setup Academic Edition for virtual translation agency, follow these steps:

  1. Create PM accounts for those who will manage projects. These can be either lecturers or students that are already familiar with basic Memsource functionality.

    To create departments within your license, use business units. This allows some projects, TMs, and TBs to be visible only to users within the selected unit.

  2. If more than the standard translation, revision, and client revision steps are required, create more workflow steps.

  3. Have PM students create user accounts for translators, reviewers, etc.. Users can have multiple roles and can be either students or educators.
