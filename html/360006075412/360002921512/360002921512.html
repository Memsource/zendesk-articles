Renaming Files

File renaming is used to automatically rename downloaded (translated) files. It can be set up in the general Setup under Project Settings (affecting only newly created projects) or for individual projects (by going to the job and selecting Edit).

The following macros are supported with the Rename completed files to option:

  • {path} - This is only for files imported in ZIP format with a folder structure.
  • {fileName} - It is also possible to remove characters from the original file name, using {filename[0:-3]} for example.
  • {sourceLang}
  • {targetLang}
  • {workflow}
  • {status}
  • {date} - the date that the file was downloaded, in the PM's timezone
  • {time} - the time that the file was downloaded, in the PM's timezone
  • {userName} - of the assigned Linguist or Vendor. If multiple Linguists are assigned, the name will be empty.
  • {replaceExt[po]} - replaces the extension of the file. For example, the original file test.POT will be downloaded as test.PO. The extension will be replaced by the one in the [ ] brackets. Please note that changing the extension will not change the format itself. That is, renaming 'docx' to 'xlsx' will not change a Word file into an Excel file.

 

Here's an example of how this works. By copying the following script:

{path}/{filename}-{sourceLang}-{targetLang}-{workflow}-{status}-{date}-{time}-{userName}

You'll get a file named: 

Completed file: testing/TestFile-en_us-ja-T-C-20160110-0912-ling1.xlf

If Rename completed files to is not selected in a project with multiple target languages, and the user decides to download multiple jobs with different targets, the downloaded ZIP file will have folders with language codes added.

Renaming Analyses

The default analysis name "Analysis #1", etc can be changed manually, or by using these macros:

  • {projectName}
  • {sourceLang} - This results in a name with the source language name added (the same as it does with file renaming).
  • {targetLang} - This results in a name with the target language added. If multiple languages are analyzed, the language will be empty.
  • {userName} - This will include the username of the assigned Linguist or Vendor. If multiple Linguists are assigned, the name will be empty.
  • {workflow}
  • {innerId}

These renaming patterns can be set up in multiple places:

  • The main Setup page (This will change the renaming patterns for the whole organization.)
  • Inside of a project
  • In a project template
  • In the Analyze dialog window (when creating a new analysis)
