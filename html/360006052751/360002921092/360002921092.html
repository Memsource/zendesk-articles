Email templates are used for the automatic notification of users. For example, notifying Linguists or Vendors about new jobs or notifying Project Managers when a job is marked as Completed. Email templates can be modified in the Setup of your account by clicking on Email Templates in the Administration section and then on the template name to be edited.

You can use Template macros in your emails and you can add CC or BCC email addresses to send a copy to other users. Below is a list of pre-made email templates available in Memsource. Please note that some are only available in the Ultimate edition.

  1. Job Assigned
    • Launched manually by a Project Manager by clicking on Tools and selecting Email... in a project, or sent automatically upon the creation of a new job that has the Notify Linguist option enabled.
    • This email is sent to the Linguist(s) or Vendor(s), giving notification regarding the availability of new translation job(s).
  2. Job Status Changed
    • Launched automatically.
    • When a Linguist or Vendor changes a job's status, this email notification is sent to the Project Manager.
  3. Job Rejected
    • Launched automatically.
    • When a Linguist sets a job status to Rejected in a project with multiple workflow steps, the Linguist in the previous workflow step is notified.
  4. Next Step Ready
    • Launched automatically.
    • When a Linguist sets a job status to Completed in a project with multiple workflow steps, the Linguist in the next workflow step is notified.
  5. Login Info
    • When a new user is added to an organization, an email will be sent to them automatically which contains their username and a link for creating a password. However, a Project Manager can also launch it manually by selecting a user in the Users page and clicking on the Email login button.
    • An email with the user's username and the login link is sent. The user then generates a new password with the link provided. 
  6. Shared Project Assigned
    • Launched automatically.
    • The Vendor will receive an email when a Project Manager shares a project with them by clicking on Share and selecting Share with Vendor in a Project.
  7. Shared Project Status Changed
    • Launched automatically.
    • When a Vendor changes a project status, an email notification is sent to the Buyer.
  8. Work completed
    • Launched automatically.
    • When a Project Manager sets the status of a project that was created using the Automation Widget to Completed, the client is notified by email with a link from which the translation can be downloaded.
  9. New project via Automation Widget
    • Launched automatically.
    • The Project Manager is notified about a project created via the Automation Widget.
  10. New project via Automation Widget (unsupported files)
    • Launched automatically.
    • When unsupported files are submitted via the Automation Widget, the Project Manager is notified.
  11. Quote Ready
    • Launched automatically.
    • This email template will be sent to customers when files are submitted via the Automation Widget.
  12. New job published on Job board
    • Launched automatically.
    • When a Project Manager publishes jobs on the Job Board (by going to Share and clicking on Publish on Job Board), an email is sent to all relevant vendors with the Receive Job Board offers option activated for their account.
  13. New response from Job board
    • Launched automatically.
    • This email is sent when a Vendor replies to a job board offering.
  14. Quote failed
    • Launched automatically.
    • When files submitted via the Automation Widget cannot be processed, this email is sent to the person who submitted them.
  15. Project transferred to buyer
    • Launched automatically.
    • The Buyer will receive an email when a Project Manager shares a project with them by clicking on Share and selecting Transfer to Buyer menu in the project.

Memsource will always use the first email template available for automated notifications.
