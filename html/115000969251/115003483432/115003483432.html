Slow response time from Memsource or the web editor can be caused either by problems on the Memsource side or by your internet connection. If you are experiencing a slow response time, there are a few things you can do:

  1. Check out our Status page to see if there are any incidents reported and to see the Web App's Response Time. If the response time is approximately 500 ms, Memsource is working correctly. If the response time is higher, please contact the Memsource Support team.


  2. You can also track the accessibility of Memsource servers across the world using Host Tracker. Open this link and run a new "Check" to get the latest results.


  3. In most cases, the problem is caused by your Internet Service Provider who could be re-routing your connection to the Memsource server via too many other servers, thus slowing the speed. If you are working with Windows, please download WinMTR—an application for tracking connection routes. You can then generate a log and discuss it with your provider.
