Renaming Files

File renaming automates the renaming of downloaded (translated) files. Access to the settings are found in the Setup ( ) page under Project Settings (affecting only newly created projects) or for individual projects by editing the specified job.

Supported macros:

  • {path} - Only for files imported in ZIP format with a folder structure.
  • {fileName} 
    • Characters can be removed from the original file name by using a negative integer {filename[0:-3]}.
  • {sourceLang} -Adds source language
  • {targetLang} - Adds target language
  • {workflow}
  • {status}
  • {date} - date of file download in the PM's timezone
  • {time} - time of file download in the PM's timezone
  • {userName} - Adds username of the assigned Linguist or Vendor. If multiple Linguists are assigned, the name will be empty.
  • {replaceExt[xx]} - The file extension is replaced by that specified within [ ] brackets.
    • Example: If {replaceExt[txt]} is specified, the file test.POT is downloaded as test.txt.
    • Changing extensions does not change the format itself; renaming 'docx' to 'xlsx' does not change a Word file into an Excel file. 

Example:

The following script:

  • {path}/{filename}-{sourceLang}-{targetLang}-{workflow}-{status}-{date}-{time}-{userName}

Produces: 

  • testing/TestFile-en_us-ja-T-C-20160110-0912-ling1.xlf

If Rename completed files to is not selected in a project with multiple target languages, and the user decides to download multiple jobs with different targets, the downloaded ZIP file will have folders with language codes added.

Renaming Analyses

Default analysis name "Analysis #1, #2, etc." can be changed manually or by using these macros:

  • {projectName}
  • {sourceLang} - Adds source language
  • {targetLang} - Adds target language. If multiple languages are analyzed, the language will be empty.
  • {userName} - Adds username of the assigned Linguist or Vendor. If multiple Linguists are assigned, the name will be empty.
  • {workflow}
  • {innerId}

Renaming patterns can be set up in multiple places:

  • The main Setup ( ) page (This will change the renaming patterns for the whole organization.)
  • Inside of a project
  • In a project template
  • In the Analyze dialog window (when creating a new analysis)
