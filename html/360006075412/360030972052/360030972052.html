Memsource supports the translation of plural-sensitive segments. These are segments which include a number. For example, an online store might have a notification saying “There are 6 items left”. However, in these kinds of segments, the number may change. If the number of items went down from 6 to 1, the word “items” would no longer be plural. Memsource supports the translation of these segments through the use of Plural Forms. Currently, this feature only supports .po files

Once apluralized string is imported into Memsource, it is shown in all its forms in separate segments. Linguists will be able to see the required plural form for the active segment by looking at the Context Note at the bottom of the Memsource Editor where the Plural Category and the Plural Example are specified. For an example of how these will be displayed, please see the screenshot in the Plural Forms in the Memsource Editor section below.

Note: Currently, the feature is only supported for .po files.

Plural Forms Set Up

To use plural forms, your .po file will need to have a formula for plural forms in the file header. You should make sure that the formula is there, otherwise, Memsource will generate it automatically based on the Unicode CLDR standard.

When setting up your files, you should use the Segment Key as the context type in the File Import Settings. It’s also important to note that only cardinal numbers are supported.

Plural Forms in the Memsource Editor

When a Linguist or Vendor selects a segment that contains a plural form, the Context Note at the bottom of the Memsource Editor will always open. The Context Note in the Tools dropdown menu will be greyed out with no option to turn it on/off. The shortcut for opening/closing the Context Note in the Tools dropdown menu will also be deactivated when a segment with a plural form is selected. 

The language locale and the plural category will be shown as part of the context for plural segments. (See the sections highlighted in blue in the screenshot below.) If you use the msgctxt line in your file, the context string will become part of the context, as the sections highlighted in red in the screenshot show.
